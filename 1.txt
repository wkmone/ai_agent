核心步骤与代码样例

假设您已经定义了一个包含多个工具方法的工具类，例如一个DateTimeTools（包含获取当前时间和设置闹钟）和一个CalculatorTools（包含一个加法计算）。

1. 定义包含多个工具的工具类

您可以使用@Tool注解在一个或多个类中声明多个工具方法。

// 工具类1：处理日期时间
import org.springframework.ai.tool.annotation.Tool;
import org.springframework.ai.tool.annotation.ToolParam;
import org.springframework.context.i18n.LocaleContextHolder;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

class DateTimeTools {
    @Tool(description = "获取用户所在时区的当前日期和时间")
    public String getCurrentDateTime() {
        return LocalDateTime.now()
                .atZone(LocaleContextHolder.getTimeZone().toZoneId())
                .toString();
    }

    @Tool(description = "为用户设置一个指定时间的闹钟，时间格式为ISO-8601 (例如: 2026-03-12T10:15:30)")
    public void setAlarm(@ToolParam(description = "ISO-8601格式的时间字符串") String time) {
        LocalDateTime alarmTime = LocalDateTime.parse(time, DateTimeFormatter.ISO_DATE_TIME);
        System.out.println("【系统日志】闹钟已设置为: " + alarmTime);
        // 这里可以替换为实际的闹钟逻辑
    }
}

// 工具类2：简单计算器
import org.springframework.ai.tool.annotation.Tool;
import org.springframework.ai.tool.annotation.ToolParam;

class CalculatorTools {
    @Tool(description = "计算两个数字的和")
    public double calculateSum(@ToolParam(description = "第一个加数") double a,
                               @ToolParam(description = "第二个加数") double b) {
        return a + b;
    }
}
2. 使用 ToolCallbacks工具类批量创建 ToolCallback数组

这是将多个工具类实例转换为ToolCallback数组的核心步骤。ToolCallbacks.from()方法会自动扫描传入对象中所有被@Tool注解的方法，并为每个方法创建一个对应的ToolCallback。
java
下载
复制
import org.springframework.ai.tool.ToolCallback;
import org.springframework.ai.tool.ToolCallbacks;

// 1. 实例化您的工具类
DateTimeTools dateTimeTools = new DateTimeTools();
CalculatorTools calculatorTools = new CalculatorTools();

// 2. 使用 ToolCallbacks.from() 将多个工具对象转换为 ToolCallback 数组
// 您可以传入多个工具实例，该方法会返回一个包含所有工具方法的ToolCallback数组。
ToolCallback[] allTools = ToolCallbacks.from(dateTimeTools, calculatorTools);

// 现在 `allTools` 数组中包含了三个ToolCallback:
// 1. 对应 `DateTimeTools.getCurrentDateTime`
// 2. 对应 `DateTimeTools.setAlarm`
// 3. 对应 `CalculatorTools.calculateSum`
3. 将 ToolCallback数组提供给模型使用

您可以将上一步得到的allTools数组提供给ChatClient或ChatModel。

方式一：通过 ChatClient使用（推荐，更简洁）
java
下载
复制
import org.springframework.ai.chat.client.ChatClient;
// ... 假设已有 ChatModel 实例 chatModel

// 创建ChatClient并传入工具数组
String response = ChatClient.create(chatModel)
        .prompt("请帮我计算一下 12.5 加 7.3 等于多少？")
        .tools(allTools) // 传入包含所有工具的ToolCallback数组
        .call()
        .content();

System.out.println(response);
当用户提问“现在几点了？”或“10分钟后提醒我”时，模型会自动调用相应的工具。

方式二：通过 ChatModel和 ToolCallingChatOptions使用

import org.springframework.ai.chat.model.ChatModel;
import org.springframework.ai.chat.prompt.Prompt;
import org.springframework.ai.model.chat.ToolCallingChatOptions;
// ... 假设已有 ChatModel 实例 chatModel

// 1. 构建包含工具配置的ChatOptions
ToolCallingChatOptions chatOptions = ToolCallingChatOptions.builder()
        .toolCallbacks(allTools) // 传入包含所有工具的ToolCallback数组
        .build();

// 2. 创建Prompt并传入选项
Prompt prompt = new Prompt("请设置一个下午3点的闹钟。", chatOptions);

// 3. 调用模型
ChatResponse chatResponse = chatModel.call(prompt);
// ... 处理响应
关键点总结
批量转换：ToolCallbacks.from(Object... toolObjects)方法的核心价值在于它能自动、批量地将一个或多个普通Java对象（其方法用@Tool标注）转换为AI模型可直接使用的ToolCallback数组。您无需为每个方法手动创建MethodToolCallback。
工具注入：无论是通过ChatClient.tools()还是ChatModel的ToolCallingChatOptions.toolCallbacks()，其本质都是将ToolCallback数组提供给模型，告知模型本次对话中“可用的工具列表”。
动态与静态：上述方式是将工具“静态”地添加到单次请求中。文档中还提到了通过ChatClient.builder().defaultTools(...)或ChatModel构造时的defaultOptions来设置默认工具，这些工具会对所有通过该Client/Model发起的请求生效，需谨慎使用。
补充说明（程序化方式）：

如果您的工具不是通过@Tool注解声明式定义的，而是通过MethodToolCallback.builder()程序化构建的，那么您需要手动将这些ToolCallback实例添加到一个List或数组中，然后传递给Client或Model。ToolCallbacks工具类在这种情况下不适用。
