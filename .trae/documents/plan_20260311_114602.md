# 前端配置大模型信息实现方案

## 1. 后端实现

### 1.1 创建模型配置实体类
- 创建 `ModelConfig` 实体类，包含以下字段：
  - id: 主键
  - name: 配置名称
  - provider: 模型提供商（如openai、anthropic等）
  - apiKey: API密钥
  - baseUrl: API基础URL
  - modelName: 模型名称
  - temperature: 温度参数
  - maxTokens: 最大令牌数
  - isDefault: 是否为默认配置
  - status: 状态
  - createTime: 创建时间
  - updateTime: 更新时间
  - deleted: 逻辑删除标记

### 1.2 创建Mapper和Service
- 创建 `ModelConfigMapper` 接口
- 创建 `ModelConfigService` 接口和实现类
- 实现CRUD操作，包括获取默认配置、设置默认配置等方法

### 1.3 创建API接口
- 创建 `ModelConfigController`，提供以下接口：
  - GET /api/v1/model-config: 获取所有配置
  - GET /api/v1/model-config/default: 获取默认配置
  - POST /api/v1/model-config: 创建新配置
  - PUT /api/v1/model-config/{id}: 更新配置
  - DELETE /api/v1/model-config/{id}: 删除配置
  - PUT /api/v1/model-config/{id}/default: 设置为默认配置

### 1.4 修改ChatService
- 修改 `ChatServiceImpl`，使用动态获取的模型配置
- 实现配置缓存机制，提高性能
- 添加配置切换逻辑

## 2. 前端实现

### 2.1 创建模型配置管理界面
- 在前端添加模型配置管理页面
- 实现配置列表展示
- 实现配置的添加、编辑、删除功能
- 实现设置默认配置功能

### 2.2 集成到聊天界面
- 在聊天界面添加模型选择下拉框
- 实时切换使用的模型
- 显示当前使用的模型信息

## 3. 数据库实现

### 3.1 创建模型配置表
- 创建 `model_config` 表，对应 `ModelConfig` 实体类
- 添加必要的索引

## 4. 功能特点

### 4.1 多模型支持
- 支持配置多个不同的模型提供商
- 支持不同的模型参数配置

### 4.2 动态切换
- 前端可以实时切换使用的模型
- 后端自动应用最新的配置

### 4.3 配置管理
- 完整的CRUD操作
- 默认配置设置
- 配置状态管理

## 5. 技术实现要点

### 5.1 后端
- 使用MyBatis Plus进行数据持久化
- 实现配置缓存，减少数据库查询
- 提供RESTful API接口

### 5.2 前端
- 使用Vue3 + Element Plus实现界面
- 实现表单验证
- 实现配置切换逻辑

### 5.3 安全性
- API密钥加密存储
- 权限控制（可选）
- 输入验证
